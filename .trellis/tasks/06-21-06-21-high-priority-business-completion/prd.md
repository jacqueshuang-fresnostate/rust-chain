# 完善高优先级未完成业务

## 目标

补齐本轮审计中标记为高优先级的未完成业务，避免 PC 端暴露无法使用的功能入口，并补上已经有页面但后端缺失的关键接口。

## 范围

- PC 找回密码接口从前端假返回改为真实后端流程。
- PC 杠杆/合约交易页移除或改成本地设置未被后端支持的操作入口。
- PC 资产页去掉假造的杠杆资产数据和无效划转入口。
- 用户中心去掉没有实际路由和后端能力的 API 管理死链。
- 顺手处理明显高风险的本地明文记住密码问题。

## 非目标

- 不新增完整 API Key 管理系统。
- 不新增杠杆资金划转、批量平仓、委托撤单等后端未建模能力。
- 不重构后台页面样式或扩大到中低优先级清单。

## 验证

- 后端执行最贴近改动的 Rust 检查。
- PC 端执行 TypeScript/构建检查中最贴近可用的一项。
