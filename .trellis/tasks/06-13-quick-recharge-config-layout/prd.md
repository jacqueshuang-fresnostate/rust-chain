# 优化快速充值配置页布局

## 背景

后台快速充值配置页字段数量不多，但原实现使用 Tabs 和紧凑表单网格，桌面端大量横向空间没有被利用，配置项视觉上仍然挤在一起。

## 目标

- 直接展开商户接口、充值资产、回调跳转三组配置。
- 使用 Semi Row/Col 响应式栅格增加字段间距和桌面横向排布。
- 保持快速充值配置保存接口、字段名和校验逻辑不变。
- 增加前端测试覆盖宽松布局和保存 payload。

## 非目标

- 不调整快速充值后端接口和订单逻辑。
- 不改变后台路由、导航和权限流程。
- 不新增自定义 CSS 文件规则。

## 验收

- 页面不再使用紧凑的 `admin-action-form` 布局。
- 三组配置同时显示，桌面端字段按 Semi 栅格展开。
- 保存配置时请求体与原接口契约保持一致。
