# 优化充值地址池批量新增和限定资产多选

## 背景

后台“添加充值地址”的原逻辑一次只能录入一个地址，并且限定资产是单文本字段。实际地址池需要支持同一网络下批量录入多条地址，同时一个地址规则可以限定多个资产；用户申请充值地址时，系统应优先分配匹配该资产限制的地址。

## 目标

- 后台新增充值地址支持多行地址一次提交。
- 限定资产使用资产下拉多选，空值表示该网络任意资产可用。
- 后端地址池保存多资产限制，并兼容旧 `asset_symbol` 单资产字段。
- 用户申请充值地址时匹配新的多资产限制。
- 列表和详情不显示资产 ID，限定资产展示为资产符号列表。

## 非目标

- 不改动已应用过的旧迁移文件 checksum。
- 不接入真实链上地址生成或充值监听。
- 不重做地址回收、禁用和审计以外的充值业务流程。

## 验收

- 后台批量新增接口可一次创建多条地址，返回创建后的地址列表。
- 多资产限定会落库到 JSON 字段；旧单资产请求仍可正常创建和查询。
- 钱包申请充值地址时可命中 `asset_symbols_json`，并优先选择匹配资产限制的地址。
- 前端添加弹窗可选择多个资产、添加多行地址并提交到批量接口。
