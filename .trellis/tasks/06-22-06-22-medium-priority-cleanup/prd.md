# 中优先级未完成项清理

## 目标

处理当前审计中标记为中优先级的 PC 端和交易记录问题，避免假分页、错误全局 WS 连接和占位页面继续暴露给用户。

## 范围

- 交易记录改为请求后端分页结果，避免固定拉取 100 条后本地分页导致漏数据。
- PC 全局布局不再默认连接 spot WS，由现货、杠杆、秒合约页面各自维护业务连接。
- OTC 占位入口隐藏，直接访问 `/otc` 时跳转到行情页。
- 清理旧 `BinaryOptions.vue` 秒合约 mock 页面，避免和真实 `SecondOptions.vue` 产生歧义。

## 非目标

- 不新增完整 P2P/OTC 业务模块。
- 不重做交易记录 UI 视觉结构。
- 不改动未列出的低优先级旧页面。

## 验证

- PC TypeScript 检查。
- 相关静态测试或路由/WS 测试。
- 后端如涉及钱包流水分页接口，执行最贴近的 Rust 检查或测试。
