# 优化合买认购记录投注内容显示

## Goal

后台“控制开奖号码 / 合买认购记录”中的“投注内容”需要从原始对象或 JSON 字符串，优化为运营可读的中文内容摘要。

## Context

- 当前仓库内未检索到独立的“控制开奖号码”“合买认购记录”路由或资源配置。
- 后台列表和详情复用 `AdminResourcePage`、`DetailDrawer` 的通用显示能力。
- 现有通用表格对未声明为 JSON 的对象值会退化为 `String(value)`，容易显示为 `[object Object]`。

## Requirements

- 后台表格遇到“投注内容”相关字段时，支持格式化 JSON 字符串、对象、数组和按位选号结构。
- 后台详情 SideSheet 遇到“投注内容”相关字段时，使用相同格式化逻辑。
- 字段 key 或列标题能识别 `bet_content`、`betContent`、`ticket_content`、`bet_info`、`投注内容` 等常见命名。
- 对常见字段使用中文标签，例如玩法、号码、位置、注数、金额、倍数。
- 不改变现有业务接口、数据库和非投注字段的展示逻辑。

## Acceptance Criteria

- [x] 表格中 `bet_content` 对象不再显示 `[object Object]`。
- [x] 详情 SideSheet 中投注内容按中文摘要显示。
- [x] JSON 字符串形式的投注内容可以被解析并展示。
- [x] 对象数组形式的投注内容按 “第 1 位：0、1、2” 这类结构展示。
- [x] 前端测试覆盖投注内容格式化和通用表格渲染。

## Out of Scope

- 不新增彩票/开奖/合买业务路由。
- 不调整后端返回字段或数据库结构。
- 不改动现有现货、秒合约、理财等业务页面。
