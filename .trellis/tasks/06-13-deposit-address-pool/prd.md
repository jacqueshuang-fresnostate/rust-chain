# 充值地址池配置与分配

## 背景

平台需要在管理后台维护充值地址池，用户充值时不是自由填写地址，而是从地址池申请一个地址。被分配的地址在回收前只能被同一个用户使用，用于避免多个用户共用充值地址造成入账归属不清。

## 范围

- 后台可以新增、查看、编辑、回收充值地址池地址。
- 地址支持网络类型：
  - ETH
  - Base（使用 ETH 地址格式，作为独立网络配置）
  - Tron
  - BTC
  - Solana
- 用户充值时按资产和网络申请地址。
- 可用地址被分配后绑定用户；绑定期间其他用户不能再获得该地址。
- 后台可以回收已分配地址，回收后该地址可再次分配。

## 数据流

后台录入地址池地址 -> 数据库存储网络、地址、状态 -> 用户充值页请求资产/网络充值地址 -> 后端查找该用户已有分配 -> 若没有则锁定并分配一个可用地址 -> 返回给用户展示 -> 后台可查看分配状态和回收地址。

## API 合约

后台：

- `GET /admin/api/v1/deposit-address-pool`：分页/筛选地址池。
- `POST /admin/api/v1/deposit-address-pool`：新增地址。
- `PATCH /admin/api/v1/deposit-address-pool/:id`：编辑备注、状态等基础信息。
- `POST /admin/api/v1/deposit-address-pool/:id/reclaim`：回收地址，清除当前用户绑定。

用户端：

- `POST /api/v1/wallet/deposit-address`：按 `asset_symbol` 和 `network` 获取或申请充值地址。

## 规则

- 地址池网络枚举统一使用小写：`eth`、`base`、`tron`、`btc`、`solana`。
- ETH 和 Base 都使用 EVM 地址格式，但作为不同网络独立分配。
- 同一网络同一地址不可重复录入。
- 地址状态：
  - `available`：可分配
  - `assigned`：已分配
  - `disabled`：禁用，不可分配
- 已分配地址只能返回给绑定用户。
- 分配时必须在数据库事务中锁定候选地址，避免并发分配给多个用户。
- 回收地址会清空 `assigned_user_id` 和 `assigned_at`，状态回到 `available`。

## UI 要求

- 后台新增“充值地址池”配置页。
- 表格展示网络、地址、状态、绑定用户邮箱、绑定时间、备注。
- 新增/编辑表单使用中文网络下拉框。
- 已分配地址显示“回收”操作。
- 用户 PC 充值页根据选择的资产和网络请求地址，并展示后端返回地址。

## 验收

- 后台可以创建 ETH/Base/Tron/BTC/Solana 地址池地址。
- 用户申请某网络充值地址时，若已有绑定则返回原地址。
- 地址被用户 A 分配后，用户 B 不能获得同一个地址。
- 后台回收后，地址可以再次分配给其他用户。
- 表格与表单字段中文化。
- 添加后端路由测试、后台资源配置测试、PC 适配层测试。
