# PC端交易页面接口与WSS对接审计

## 背景

需要查看 PC 端合约、现货、秒合约交易页面，确认页面使用的 HTTP API 与 WebSocket 订阅是否已经对接到当前 Rust 后端，找出仍然是占位、旧接口、未接后端或页面调用但后端不支持的功能。

## 范围

- 现货页面：`/spot/:symbol?`，页面、交易表单、当前/历史委托、盘口、成交、K 线。
- 合约页面：`/contract/:symbol?`，页面、开平仓表单、持仓/委托、盘口、ticker、成交、K 线。
- 秒合约页面：`/second/:symbol?`，页面、产品/周期、下单、持仓/历史、盘口、ticker、成交、K 线。
- WebSocket：核对 `spot`、`margin`、`seconds` 三类业务订阅是否被页面实际使用，并记录后端端点现状。

## 非范围

- 本任务不直接修改业务逻辑和 UI。
- 本任务不处理后台页面、充值、闪兑、新闻等非交易页问题。
- 本任务不做真实外部行情源联调，只依据代码链路和本地类型检查/搜索验证。

## 交付

- 输出审计结论：每个页面哪些已对接、哪些未对接、哪些属于后端能力缺口。
- 在任务目录沉淀审计报告，便于后续按缺口拆分修复任务。
