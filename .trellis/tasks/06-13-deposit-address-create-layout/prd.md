# 优化添加充值地址页面

## 背景

后台充值地址池支持一次添加多行地址，并可限定多个资产。当前创建 SideSheet 将网络、资产、状态和地址明细堆叠在同一个表单里，批量录入时不够清晰。

## 目标

- 使用 Semi UI 默认布局能力重新组织添加充值地址 SideSheet。
- 将全局规则与地址明细分区展示。
- 保留网络、限定资产多选、初始状态、多行地址录入、Memo/Tag、备注和确认提交能力。
- 提交接口与请求结构保持兼容。

## 非目标

- 不调整充值地址池后端接口。
- 不修改编辑、回收或列表筛选逻辑。
- 不引入新的自定义视觉体系。

## 验收标准

- 打开“添加充值地址”后，表单展示为清晰分区。
- 可添加多行充值地址，每行可填写地址、Memo/Tag、备注，并可删除多余行。
- 限定资产仍为多选下拉。
- 提交后仍调用 `/admin/api/v1/deposit-address-pool/batch`，请求体字段不变。
- 相关前端测试与类型检查通过。
