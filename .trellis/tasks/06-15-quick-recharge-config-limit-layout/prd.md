# 后台快速充值配置限制结构优化

## 背景

后台“快速充值配置”页面当前把法币币种、到账资产、收款网络、单笔最小金额、单笔最大金额都放在“充值资产”同一组里，视觉层级偏混乱。用户希望优化限制配置结构，让页面更容易理解和维护。

## 目标

- 优化 `QuickRechargeConfigPage` 中快速充值限制相关字段的展示结构。
- 将币种、到账资产、收款网络与单笔金额限制拆成清晰分组。
- 保持现有读取、保存、启停、测试联通接口逻辑和请求字段不变。
- 使用现有 Semi UI 组件与页面本地布局模式，不引入新的业务抽象。

## 范围

- 后台快速充值配置页面布局。
- 该页面对应的前端测试。
- 任务进度记录。

## 不在范围

- 不修改后端快速充值接口。
- 不调整 GMPay 回调、测试订单或支付跳转逻辑。
- 不改动其它后台页面的统一表格或筛选样式。

## 验收标准

- 页面中限制配置拆分为更清楚的“入账范围”和“单笔金额限制”结构。
- 原有字段仍可编辑，保存 payload 与现有接口字段保持一致。
- 页面测试覆盖新结构并通过。
- TypeScript 类型检查通过，触碰文件无空白 diff 问题。
