# 配置 PC 未登录页面状态

## Goal

梳理 PC 端需要登录后才能查看或操作的页面，在未登录时展示明确的页面状态和登录入口，避免空白、误报加载失败、401 请求或直接跳转造成的体验断层。

## Requirements

* 用户中心下的个人资产、安全、KYC、交易记录、订单等页面应在未登录时显示统一的登录提示状态。
* 公共页面中的登录后操作区域（如竞猜下注、贷款申请、闪兑资产余额等）应避免未登录时直接请求用户私有接口。
* 未登录状态需要有清晰文案和登录按钮，并保留当前页面作为 redirect。
* 不改变已登录状态的数据加载和提交逻辑。

## Acceptance Criteria

* [x] 扫描 PC 端会调用用户私有接口的页面。
* [x] 用户中心页面未登录时不再触发私有 API 请求，并显示登录状态。
* [x] 关键业务页面未登录时显示明确操作状态或登录入口。
* [x] PC type-check 和相关静态测试通过。

## Definition of Done

* 相关 PC 测试和类型检查通过。
* `docs/superpowers/PROGRESS.md` 记录本次交付。
