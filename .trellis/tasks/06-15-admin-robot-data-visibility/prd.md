# 后台机器人数据默认隐藏开关

## 背景

后台现货订单列表已经有“显示机器人订单”开关，但开关位置不够合适；其它会混入系统机器人用户数据的后台列表也需要默认隐藏机器人数据，并允许管理员手动开启查看。

## 目标

- 将“显示机器人订单”开关移动到更合适的表格工具区域，而不是普通筛选输入区。
- 后台所有可能包含机器人用户数据的列表默认不显示机器人数据。
- 对这些列表提供明确开关，管理员开启后可以查看机器人数据。
- 保持现有筛选和分页行为不被破坏。

## 范围

- 管理后台资源页筛选/工具栏布局。
- 现货订单列表。
- 钱包账户、钱包流水等会通过用户邮箱显示内部机器人账户数据的后台列表。
- 后端相关列表查询参数。
- 前后端测试。

## 非目标

- 不删除机器人数据。
- 不改变机器人撮合/流动性业务逻辑。
- 不改变用户端接口。

## 验收标准

- 现货订单列表默认请求不带或关闭 `include_internal`，默认隐藏机器人订单。
- “显示机器人订单”开关显示在表格工具区的合适位置。
- 其它包含机器人用户数据的后台列表默认隐藏机器人数据，并提供同类开关。
- 开启开关后对应请求带上 `include_internal=true`。
- 后端列表接口默认过滤 `@internal.local` 机器人用户数据，显式开启时返回。
