# 添加充值地址导入

## 背景

后台充值地址池创建页已支持手动新增多行地址。运营批量维护地址时逐行输入效率低，需要可以通过导入文件快速生成多条地址明细。

## 目标

- 在“添加充值地址”SideSheet 中新增导入入口。
- 支持导入 `.csv` / `.txt` 文件。
- 文件每行解析为一条地址，字段顺序为：充值地址、Memo/Tag、备注。
- 导入后填充当前批量地址明细，继续沿用现有网络、限定资产、初始状态和提交接口。
- 导入失败或空文件时给出中文错误提示。

## 非目标

- 不新增后端导入接口。
- 不改变 `/admin/api/v1/deposit-address-pool/batch` 请求结构。
- 不调整地址池列表、编辑或回收逻辑。

## 验收标准

- 管理员可以在添加充值地址弹窗中点击“导入地址”选择 CSV/TXT 文件。
- 有效文件导入后，地址明细自动展示多行地址。
- 支持逗号、制表符或竖线分隔；空行会被忽略。
- 提交时请求体 entries 与导入内容一致。
- 相关前端测试、类型检查和 diff 检查通过。
