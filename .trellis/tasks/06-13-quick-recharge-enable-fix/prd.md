# 修复快速充值无法启用

## 背景

后台 GMPay 快速充值配置页在开启开关后，会根据前端条件直接禁用“保存快速充值配置”按钮。管理员如果未一次性补齐 API 地址、PID、Secret 和回调地址，会感觉快速充值无法启用，并且页面没有给出明确缺失项。

## 目标

- 快速充值配置页在开启开关后仍允许点击保存动作。
- 启用前缺少必要字段时，页面用中文明确提示缺少哪些字段。
- 确认保存时如果启用必要字段缺失，前端阻止提交并显示中文错误，不发起无效 PATCH。
- 配置完整时可以保存 `enabled: true`。

## 非目标

- 不修改 GMPay/Epusdt 后端签名、下单或回调入账逻辑。
- 不改变快速充值数据库结构。

## 验收

- 后台快速充值未启用状态可切换开关并看到缺失项。
- 保存按钮不会因缺少启用字段直接不可点击。
- 完整配置开启后 PATCH payload 包含 `enabled: true`。
- 相关前端测试和类型检查通过。
