# 提现梯度手续费配置

## Goal

支持每个可提现资产单独配置按提现金额区间计算的手续费，例如 USDT 提现金额 `1-100` 收取 `1%`，`100-500` 收取 `2%`。

## Context

- 资产表已经有 `withdraw_enabled` 和固定 `withdraw_fee`。
- 用户提现请求由后端 `/api/v1/wallet/withdrawals` 创建，当前服务端按资产读取固定手续费。
- PC 提现页会展示手续费并提交 fee 字段，但实际扣费必须以服务端配置为准。
- 后台资产管理已有资产创建/修改能力，需要在同一配置面支持提现手续费梯度。

## Requirements

- 可提现资产可以配置多个提现手续费区间。
- 每个区间包含最小金额、最大金额和手续费比例（百分比）。
- 最大金额为空表示该区间无上限。
- 区间按“最小值包含，最大值不包含”匹配，避免相邻区间在边界金额重复命中。
- 未配置梯度或金额不命中区间时，继续使用资产固定 `withdraw_fee` 作为兜底。
- 后端创建提现申请时按当前提现金额计算手续费，并写入提现申请。
- 用户端提现页根据输入金额实时预估手续费。
- 后台资产创建/修改可以维护梯度配置，列表能看懂该资产是否配置了梯度提现手续费。

## Acceptance Criteria

- [x] 数据库新增资产提现手续费梯度配置，不修改历史迁移。
- [x] 后端按资产、提现金额计算梯度手续费，固定手续费兜底不破坏。
- [x] PC 提现页可以显示按金额变化的手续费。
- [x] 后台资产配置支持新增、编辑、删除多行梯度规则。
- [x] 后端、PC、后台测试覆盖本次改动的核心路径。

## Out of Scope

- 不调整提现审核、放款、钱包流水审核逻辑。
- 不新增独立提现手续费资源页。
- 不改变充值手续费或提现开关语义。
