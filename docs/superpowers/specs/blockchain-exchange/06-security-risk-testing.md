# 06. 风控、安全、合规边界与测试验收

## 1. 风控规则

| 场景 | 规则 |
|---|---|
| 注册/登录 | IP 频率限制、设备指纹预留、异常登录提醒 |
| 下单 | 最小下单额、价格偏离限制、数量精度、频率限制 |
| 撤单 | 高频撤单限制、异常撤单监控 |
| 资产 | 余额变动必须有流水，禁止无来源改余额 |
| 新币申购 | 仅发行阶段开放；限制申购额度、重复申购和资金不足 |
| 新币认购 | 仅 listed 状态且后台开启后开放；认购所得继承币种解禁规则 |
| 新币派发/认购 | 派发和认购必须幂等；获配、认购、退款、锁定、矿工费、解禁均写资产流水 |
| 解禁矿工费 | 可按市值或收益计费；支付币种可配置；未支付时不能解禁 |
| 新币锁定仓位 | fixed_time 按 user_id + asset_id + unlock_at 聚合；relative_period 按来源订单拆分；wallet_accounts.locked 必须与活跃锁定仓位汇总一致 |
| 闪兑 | 报价 TTL、quote_id 幂等、余额复核、限额、库存、汇率偏离校验 |
| 提现 | 一期预留；二期支持地址白名单、审核、冷热钱包 |
| 后台操作 | 敏感操作二次确认、RBAC、全量审计日志 |
| 策略行情 | 仅内部交易对可用，限制价格跳变和参数越界 |
| 代理后台 | 只读团队数据，禁止修改用户资产、订单、行情策略 |

## 2. 安全设计

| 层级 | 设计 |
|---|---|
| 密码 | Argon2 或 bcrypt 哈希 |
| 用户认证 | JWT access token + refresh token |
| 后台认证 | 独立管理员登录体系 + RBAC |
| 代理认证 | 独立 agent admin 登录体系 |
| API | 请求限流、参数校验、统一错误码 |
| WebSocket | 公共频道限流，私有频道鉴权 |
| MySQL | 关键字段索引、事务控制、资产变更行级锁 |
| MongoDB | K 线唯一索引、行情归档 TTL、按 symbol 与时间建立查询索引 |
| Redis | 设置 TTL，避免无界增长；补偿任务使用分布式锁 |
| RabbitMQ | 事件幂等 key、失败重试、死信队列 |
| 审计 | 后台、新币策略、闪兑配置、代理归属变更必须审计 |

## 3. 合规边界

外部行情交易对：

- BTC/USDT、ETH/USDT 等主流币只展示 Bitget / HTX 行情。
- 不允许后台控制外部真实行情。

新币交易对：

- 使用内部策略行情。
- 新币在预热、发行、派发阶段不展示交易 K 线。
- 新币到达上市时间且交易对启用后，K 线开始生成和展示。
- 一期可生成 ticker、trade、K 线、成交量模拟。
- 系统必须区分 external、internal、strategy 市场。

上市后认购与解禁矿工费：

- 认购仅允许在新币 listed 且后台开启认购时开放。
- 认购价格来源必须明确：当前行情价或后台认购价，并写入认购记录。
- 认购所得必须继承币种解禁规则，不能绕过 locked 余额。
- 启用解禁矿工费时，未支付矿工费不能释放 locked；计费基础、费率和支付币种必须可配置并审计。

后台行情策略：

- 仅用于新币或内部测试交易对。
- 启停和参数修改必须审计。
- 必须经过 Risk Guard 校验。

闪兑：

- 主流币闪兑基于外部行情和平台点差报价。
- 新币闪兑基于后台配置汇率或浮动规则。
- locked 余额不能参与闪兑。
- 新币未上市时默认不能闪兑，除非后台单独开启。
- 汇率、手续费、限额、开关修改必须审计。

代理后台：

- 代理只能查看和统计归属用户。
- 代理不具备平台总后台权限。
- 代理不能修改闪兑汇率、手续费、限额或开关。
- 返佣结算必须保留流水和审核状态。

## 4. 全局时间字段验收

- 项目所有涉及时间的对外字段必须使用时间戳语义，默认采用 Unix milliseconds 数字。
- API、WebSocket、RabbitMQ 事件、Redis 缓存和 MongoDB 行情文档不得输出本地化时间字符串。
- Rust/MySQL 内部时间可使用 `DateTime<Utc>` / `TIMESTAMP(6)`，但跨系统边界必须转换为时间戳。
- 每个新增接口、事件、缓存 payload 和文档字段都必须在测试或评审中检查该要求。

## 5. 测试设计

| 类型 | 覆盖内容 |
|---|---|
| 单元测试 | 价格精度、下单校验、资产冻结/解冻、邀请归属计算 |
| 集成测试 | 下单、撮合、成交、资产流水完整链路 |
| 行情测试 | Bitget / HTX adapter 标准化、断线重连、主备切换 |
| 新币生命周期测试 | 预热不可申购、发行可申购、派发到账、上市后 K 线出现、锁定与解禁正确 |
| 新币锁定仓位测试 | fixed_time 聚合锁定、relative_period 按订单拆分、来源追踪、wallet_accounts.locked 与活跃仓位汇总一致、释放幂等 |
| 新币认购与矿工费测试 | listed 后认购、后台认购开关、认购锁定、按市值/收益计费、支付币种、未支付不可解禁、支付后释放、矿工费幂等 |
| 策略行情测试 | 目标价生成、波动率限制、K 线聚合、越界拒绝、停机后缺口补偿 |
| 闪兑测试 | 主流币报价、新币后台汇率、报价过期、重复确认、余额流水、locked 不可兑换 |
| 权限测试 | 用户、平台后台、代理后台权限隔离 |
| 安全测试 | JWT 过期、重复请求、限流、敏感操作审计 |
| 数据一致性测试 | 钱包余额与流水汇总一致、订单状态一致 |
| WebSocket 测试 | 订阅、取消订阅、私有频道鉴权、断线恢复 |
| 时间戳边界测试 | 所有 API、WebSocket、RabbitMQ、Redis、MongoDB 对外时间字段必须为 Unix milliseconds 时间戳 |

## 6. 一期验收标准

| 模块 | 验收条件 |
|---|---|
| 用户 | 可注册、登录、获取个人信息 |
| 资产 | 可查看账户；下单冻结、撤单解冻、成交结算正确 |
| 行情 | 可接入 Bitget / HTX；Redis 有实时 ticker / K 线缓存；MongoDB 可保存行情归档 |
| K 线 | 支持 1m、5m、15m、1h、1d 查询；历史 K 线从 MongoDB 读取 |
| 现货 | 可下限价单、市价单、撤单、查看订单与成交 |
| 闪兑 | 主流币可按外部行情报价成交；新币可按后台汇率报价成交；过期和重复报价被拒绝；流水完整 |
| 新币 | 后台可配置预热、发行申购、派发、上市、上市后认购、解禁规则、锁定仓位、解禁矿工费和策略行情；fixed_time 锁定聚合正确，relative_period 锁定按订单拆分；wallet_accounts.locked 与活跃锁定仓位汇总一致；上市后 K 线出现；认购所得按解禁规则处理；未支付矿工费不能解禁；服务重启后可补齐策略 K 线缺口 |
| 平台后台 | 可管理用户、币种、交易对、策略、审计 |
| 代理 | 可创建代理、邀请码；用户注册绑定；代理后台查看团队用户 |
| 权限 | 用户端、平台后台、代理后台权限隔离 |
| 审计 | 敏感后台操作可追溯 |
| 稳定性 | 行情断线可重连；RabbitMQ 消费失败可重试 |
| 时间字段 | 对外时间字段统一为 Unix milliseconds 时间戳，内部 `DateTime<Utc>` / `TIMESTAMP(6)` 不直接暴露 |
